# X-Perl UnitFrames — Sirus.su fork

**Версия:** 3.1.0  
**Автор форка:** [Sesqiwake](https://github.com/)  
**Оригинал:** Zek (X-Perl UnitFrames)  
**Клиент:** World of Warcraft 3.3.5a (Interface `30300`), [Sirus.su](https://sirus.su)

Замена стандартных unit-фреймов: игрок, питомец, группа, цель, цель цели, фокус и рейд. Форк адаптирован под клиент и лаунчер Sirus.su: исправления после обновы лаунчера, профили раскладки и доработка тултипов бафов.

---

## Установка

1. Скопируйте в `World of Warcraft Sirus/Interface/AddOns/` все папки `XPerl*` из репозитория (ядро, опции, рейд-фреймы и остальные модули).
2. В игре включите нужные модули X-Perl (минимум **XPerl** + **XPerl_Options** + **XPerl_RaidFrames** для рейда).
3. Зайдите в игру

Команды: `/xperl` — настройки, `/xperl lock` — заблокировать фреймы.

## Что нового в 3.1.0

Кратко — пять основных доработок этого реворка.

### 1. Профили раскладки

Именованные профили вместо старого «Слой фреймов», который сохранял **только позиции**.

**Что сохраняется в профиль:**

| Категория | Параметры |
|-----------|-----------|
| Позиции | Все основные фреймы (игрок, цель, фокус, группа, заголовки рейда 1–10, якоря мониторов и т.д.) |
| Масштаб | player, pet, target, focus, party, raid и связанные фреймы |
| Рейд | scale, spacing, anchor, сортировка по классу/алфавиту, **видимые группы 1–8**, слои по классам |
| Группа | scale, режим smallRaid |

**Где пользоваться:**

- Вкладка **«Профили»** в `/xperl` — сохранить, применить, удалить, авто-применение при входе.
- Список профилей **общий на клиент** (`XPerlConfigNew.profiles.list`); у каждого персонажа свой активный профиль и флаг авто-применения.
- **Миникарта:** наведение на иконку X-Perl — выпадающий список профилей; ПКМ — полное меню с подменю «Профили».

Старые сохранения из per-character хранилища подхватываются автоматически при первом запуске.

### 2. Источник бафов в тултипах

В тултипе бафа/дебафа снова показывается строка **«Источник: ИмяИгрока»** (опция *Global → Buffs → Caster Names* / «Имя произносителя»).

После обновы с лаунчера перестало работать из‑за:

- `UnitAura()` — неверная передача фильтра `HELPFUL` / `HARMFUL`;
- поле `caster` стало приходить как **GUID** (`Player-0-…`), а не unit-токен (`raid3`).

Исправлено: корректный вызов API, разбор GUID через `UnitTokenFromGUID`, значение по умолчанию `buffs.names = 1`.

---

### 3. Исправлены описания бафов в тултипах

Починены тултипы в **Buff Helper** (список «кто без бафа») и в хуках `SetUnitBuff` / `SetUnitAura`: дублирование строк, неверная ссылка на объект тултипа, лишний вывод кастера при вызове через сторонние обёртки (Utopia и т.п.).

---

### 4. Иконки групп рейда при смене слоёв

При переключении **слоя раскладки** (разблокировка/блокировка фреймов, смена видимых групп или классовых слоёв) заголовки рейда не обновляли иконки и подписи.

Теперь при `XPerl_UnlockFrames` / `XPerl_LockFrames` вызывается `XPerl_RaidTitles()` (и `XPerl_RaidPets_Titles()`), заголовки групп синхронизируются с текущими настройками.

---

### 5. Загрузка рейд-фреймов и просадка FPS

**Симптом:** после обновы с лаунчера в рейде видны только **заголовки групп** без игроков; при включённом X-Perl сильно падает FPS.

**Причины:**

- после `RAID_ROSTER_UPDATE` не вызывался повторный показ secure-заголовков `XPerl_Raid_Grp1`–`8`;
- при входе в рейд / `/reload` иногда не срабатывал `XPerl_Raid_ChangeAttributes()` (клиент не всегда шлёт `RAID_ROSTER_UPDATE`);
- опечатка в **XPerl_Party**: событие `RAID_ROSTER_UPDATED` вместо `RAID_ROSTER_UPDATE`;
- лишний спам `EXTRA EVENT` и нагрузка OnUpdate на «пустых» заголовках.

**Исправлено:** принудительное обновление атрибутов рейда, подписка на `GROUP_ROSTER_UPDATE`, исправление события в Party, убран лишний шум в обработчиках.

## Модули

| Папка | Назначение |
|-------|------------|
| `XPerl` | Ядро, профили, миникарта, тултипы |
| `XPerl_Options` | Окно настроек |
| `XPerl_Player`, `XPerl_Target`, `XPerl_TargetTarget`, `XPerl_PlayerPet`, `XPerl_Party`, `XPerl_PartyPet` | Unit-фреймы |
| `XPerl_RaidFrames` | Рейд-фреймы |
| `XPerl_RaidHelper`, `XPerl_RaidMonitor`, `XPerl_RaidAdmin`, `XPerl_RaidPets` | Рейд-инструменты |
| `XPerl_ArcaneBar`, `XPerl_PlayerBuffs`, `XPerl_CustomHighlight`, `XPerl_Tutorial` | Доп. модули |
